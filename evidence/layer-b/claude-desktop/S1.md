## Final answer (model's prose synthesis, verbatim)

> All three notes pinned. Here's the map of how the Dolma 3 corpus is sharded and stored across the layers in use.
>
> **The corpus and how it's sharded.** The source of truth is the Dolma 3 deduplicated corpus produced by SOC-127: ~6T tokens / ~1.26B documents, deduplicated down from the ~9T `allenai/dolma3_pool` upstream. It is partitioned into ~58,621 shards, each a `.jsonl.zst` (zstandard-compressed JSONL) file of ~50–100 MB, containing JSON lines with `id`, `text`, and `metadata`. Shards split across two top-level prefixes — `soc127/phase1_pool_shared/` for pool-derived sources (common_crawl, olmocr, …) and `soc127/phase2_nonpool_final/` for the rest (stack_edu, arxiv, finemath, wiki) — and further partitioned by `source=<src>/part=NNN/`. This is documented in `DATA_INVENTORY.md` and `soc127_modal_handoff/02_execution_spec.md` §6.1.
>
> **Storage layers.**
>
> 1. **Cloudflare R2 — primary, durable, canonical.** Bucket `soc127-dedup` at endpoint `https://0934ab8e84ac8f4e81decaf3eb121337.r2.cloudflarestorage.com`. Three logical datasets share this bucket: the corpus shards under `soc127/…`, the SOC-91 WebOrganizer parquet sidecars under `soc91-labels/` (24 topics × 24 formats = 576 bins) with stats under `soc91-stats/`, and the SOC-95 per-shard manifest under `soc95-manifest/data/` (~213 GB, one parquet per shard, columns `doc_id`, `token_count`, `shard_path`, `weborganizer_topic`, `weborganizer_format`). Materialized samples for PACE sync live at `soc134-samples/sample_<N>_docs/`. R2 was chosen as the primary plane explicitly because Modal Volume v1 is "not suitable for heavy concurrent multi-terabyte output production" (per `soc127_modal_handoff/02_execution_spec.md`). Credentials: 1Password locally via `scripts/bootstrap/with_r2_credentials.sh`, and the `r2-credentials` Modal secret in-cloud. Confirmed from `DATA_INVENTORY.md`, `WORKING_SAMPLE_DATA_ACCESS.md`, `soc127_modal_handoff/02_execution_spec.md`, `ATTRIBUTION_RUNBOOK.md`.
>
> 2. **Modal persistent volumes (`eilab-gt` workspace) — warm cache for Modal workers.** Seven volumes, all derivative of R2. `soc134-corpus-cache` (~5 TB) mirrors the R2 key tree so `shard_path` values resolve directly inside Modal functions; `soc134-manifest-cache` (~213 GB) mirrors the SOC-95 manifest; and five `soc134-output-*` volumes hold materialized per-sample outputs (sample_500/1000/5000/10000_docs plus preconditioner_100k). Output layout is `{run_id}/manifest.parquet + sample_contract.json + bin_summary.csv + worker_0000…worker_0127/*.jsonl.zst` with `.stats.json` and `.done` sentinels, plus an `.hf_upload/` directory tracking batch upload completion. Current run IDs (2026-03-20 / 2026-03-24) are pinned in the memory note. A legacy `soc134-materialize-cache` hit the 500K inode limit during a failed multi-sample run and is marked safe-to-delete. Confirmed from `DATA_INVENTORY.md` and `WORKING_SAMPLE_DATA_ACCESS.md`.
>
> 3. **Local caches — three flavors, none authoritative.**
>    - **PACE ICE scratch** (`~/scratch/tracstar/`): where the attribution / TracStar pipeline reads from. Synced down from R2 prefix `soc134-samples/sample_<N>_docs/` via `scripts/attribution/sync_sample.py`, then reorganized round-robin into 16 plain JSONL shards at `runs/tracstar/shards/<label>/shard_NNNN.jsonl` (metadata stripped to `id` + `text`). Source: `ATTRIBUTION_RUNBOOK.md`.
>    - **Local HuggingFace cache** (`.hf_cache/datasets/allenai__dolma3_mix-150B-1025`): used by the enrichment pipeline (`python -m dolma.cli enrich-format`) on PACE Slurm. 6,081 shards observed under `data/<category>/shard_*.jsonl.zst`, with per-shard metadata under `.cache/huggingface/download/…/*.metadata`. Note this caches the upstream 150B mix — it's independent of the SOC-127 6T R2 corpus. Source: `DOLMA_ENRICHMENT.md`.
>    - **Repo `data/samples/`**: metadata-only — `working_sample_manifest.parquet` + `sample_contract.json` + `bin_summary.csv` for each sample size (14 MB → 280 MB). No document text. Useful for `doc_id → shard_path / bin` lookups without any cloud access. Source: `DATA_INVENTORY.md`, `WORKING_SAMPLE_DATA_ACCESS.md`.
>
> **Data flow summary.** R2 is the only durable plane. The Modal volumes are a read-cache (corpus + manifests) and a per-sample write-sink that is then promoted to HuggingFace under `HCAI-Lab/dolma3_6T_sample_<N>_docs` (and `…_preconditioner_100k`). PACE scratch and `.hf_cache` are workflow-specific staging areas — attribution and enrichment respectively — and `data/samples/` in the repo is a thin in-tree index for offline lookups.
>
> The three pinned notes (`/memory/dolma3-storage-layer-r2.md`, `/memory/dolma3-storage-layer-modal-volumes.md`, `/memory/dolma3-storage-layer-local-caches.md`) each carry the layer-specific paths, run IDs, and source-file citations so a future session can resume without re-walking `/docs`.
