## Ask vs Agent finding (Cursor-specific)

Free-plan capacity throttling prevented loading new Agent-mode chats
on first attempt. User tried "Ask" mode as a workaround. **Ask mode
did not execute MCP tool calls** — confirmed empirically before
running S2. Agent mode resumed working once capacity cleared and was
used for the actual S2 run. This is a real adoption-friction
observation: an MCP server's reach is bound to Cursor's agentic
modes (`Agent`, `Plan`, `Debug`, `Multitask`), not the conversational
`Ask` mode. Worth documenting for users on rate-limited tiers — if
Agent is throttled, the MCP isn't reachable until it clears.

## C2 implications

**The C2 chain is partially intact on Cursor.** Three good things and
one honest gap:

✓ The model used `ls -la /memory` as instructed and saw all 3 S1
notes with correct byte counts. The directory survived across
sessions and the listing query worked through MCP. This is the
**substrate-level C2 evidence**: rows persisted, the read path is
plumbed, fresh-chat access works.

✓ The model acknowledged the prior pins explicitly in its final
answer ("*From an earlier session:* ..."), demonstrating it knew
they existed and had factored them into its mental model of what
the user was building.

✓ Three new pins landed cleanly, growing the chain 3 → 6 rows.

✗ **The model did NOT `cat`-read the S1 notes' contents.** Opus on
Claude Desktop did a batched `cat` of all three S1 notes and
explicitly carried their R2/Modal/local-caches content forward into
S2's reasoning ("we already know the R2 corpus is at
`soc127-dedup`..."). Cursor's model listed the prior notes by name
but did not open them. This means C2 *integration* — the strongest
form of the cross-session memory claim — is not evidenced on this
Cursor S2 run.

The honest read on C2 status: the **substrate works** (rows
persisted, `ls /memory` returned them), the **awareness works** (the
model knew they were there), but the **integration depth** is
weaker than the Claude Desktop chain. Whether this is a model-class
limitation (Sonnet < Opus on read-then-synthesize) or a system-prompt
issue (Cursor doesn't bias toward reading discovered files) is worth
testing in S3, where synthesis is the load-bearing requirement.
